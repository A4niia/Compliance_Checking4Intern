\chapter{Implementation}
\label{ch:implementation}

This chapter describes the technical implementation of the policy formalization pipeline.

\section{System Architecture}

The system consists of four main components:

\begin{enumerate}
    \item \textbf{Document Processor}: PDF extraction and preprocessing
    \item \textbf{LLM Classifier}: Rule identification using Ollama
    \item \textbf{FOL Formalizer}: Logic representation generation
    \item \textbf{SHACL Translator}: Validation constraint generation
\end{enumerate}

\section{Document Processing}

\subsection{PDF Extraction}

\begin{lstlisting}[language=Python]
import fitz  # PyMuPDF

def extract_text(pdf_path):
    doc = fitz.open(pdf_path)
    text = ""
    for page in doc:
        text += page.get_text()
    return text
\end{lstlisting}

\section{LLM Integration}

\subsection{Ollama API}

\begin{lstlisting}[language=Python]
import requests

def query_ollama(prompt, model="mistral"):
    response = requests.post(
        "http://localhost:11434/api/generate",
        json={
            "model": model,
            "prompt": prompt,
            "stream": False,
            "options": {"num_predict": 1024}
        }
    )
    return response.json()["response"]
\end{lstlisting}

Robust parsing with multiple fallback strategies handles malformed LLM output.

\section{SHACL Translation}

\subsection{Translation Algorithm}

\begin{lstlisting}[language=Python]
def make_shape(rule):
    fol = rule["fol_formalization"]
    predicates = extract_predicates(fol["deontic_formula"])
    
    shape = f"""
    ait:{rule_id}Shape a sh:NodeShape ;
        sh:targetClass ait:{main_class} ;
        sh:severity {severity} ;
    """
    
    for pred in predicates:
        shape += f"""
        sh:property [
            sh:path ait:{pred} ;
            sh:minCount 1 ;
        ] ;
        """
    
    return shape
\end{lstlisting}
